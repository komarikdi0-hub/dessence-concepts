# Концепции продукта dEssence

## Что нужно сделать

Предложить 3-5 визуальных концепций для приложения dEssence. Каждая концепция — это отдельный визуальный стиль («скин») для одного и того же интерфейса.

Фокус на визуале: тени, шрифты, глубина, цвета, типографика, стиль карточек, отступы, фоны, разделение зон. Логику и UX-flow продумывать не нужно.

---

## Что за продукт

dEssence — персональная AI-память. Пользователь сохраняет контент из разных источников (ссылки, фото, сообщения, скриншоты), а потом общается с AI, который использует этот контент как контекст для ответов.

**Аудитория:** обычные люди (не tech-гики), 25–45 лет, активная жизнь — рестораны, путешествия, покупки, рецепты, концерты.

**Позиционирование:** персональный AI-ассистент, который знает тебя и помогает в повседневных решениях.

---

## Layout

Desktop first. Два основных элемента на экране одновременно:

```
┌────────────────────────────────┬──────────────────┐
│                                │                  │
│         PINBOARD               │      CHAT        │
│   (визуальная сетка            │   (диалог        │
│    карточек с контентом)       │    с AI)         │
│                                │                  │
│   Карточки разных типов:       │   Вопрос ->      │
│   - ссылки (с превью)          │   Ответ AI       │
│   - фото                       │                  │
│   - текстовые заметки          │                  │
│   - видео                      │                  │
│                                │                  │
└────────────────────────────────┴──────────────────┘
```

- Pinboard занимает ~60–65% ширины, остальное — chat
- Карточки без жёсткого grid (разных размеров)

## Типы карточек в Pinboard

| Тип | Что содержит | Пример |
|---|---|---|
| Ссылка | Превью сайта + заголовок + описание + источник | Рецепт с Minimalist Baker |
| Фото/скриншот | Изображение + подпись + дата | Фото блюда из ресторана |
| Текст | Заметка/сообщение + автор + дата | «Emily прислала: попробуй этот салат» |
| Видео | Превью + заголовок + источник | YouTube-ролик с рецептом |
| Товар | Фото + название + цена + магазин | Сковородка с Amazon, $45 |

## Проблемы текущей версии

Текущий визуальный стиль вызывает негативную реакцию:

- **«Скучно»** — нет визуальной привлекательности, не хочется возвращаться
- **«Куце»** — выглядит бедно и незавершённо
- **«Непривлекательно»** — нет wow-эффекта, отсутствует ощущение качественного продукта
- **«Мусорка»** — pinboard выглядит как свалка серых блоков
- **«Ничего не изменилось»** — предыдущие итерации не решили проблему визуала

Проблема именно в визуальном стиле («обёртке»), а не в расположении элементов или логике.

## Референсы

### Pinterest
- Визуальная плотность и разнообразие карточек
- Masonry grid (карточки разной высоты)
- Ощущение «живого» контента
- Цветовое богатство за счёт самого контента

### MyMind (mymind.com)
- Минималистичный, но «глубокий» визуал
- Тени, глубина, многослойность
- Цветовое разделение зон
- Ощущение premium-продукта
- Тёплая, приятная палитра

### Fabric (fabric.so)
- Чистый layout без перегруженности
- AI sidebar рядом с контентом
- Современный, tech-forward стиль
- Хорошая работа с типографикой

### Другие
- https://raindrop.io
- http://granola.ai
- https://wise.com

## Что НЕ нужно делать

- Не продумывать UX-логику и flow (layout зафиксирован)
- Не делать wireframes или прототипы
- Не проектировать мобильную версию (только desktop)
- Не делать дизайн-систему

---

## Что нужно в результате

Для каждой из 3–5 концепций:

1. Один экран — pinboard + chat вместе, заполненный реальным контентом (10–15 карточек разных типов + 3–4 сообщения в чате) с выбранной визуальной стилистикой
2. Выбор цветовой палитры и типографики

Контент для карточек: рецепты, рестораны, одежда, путешествия, книги, концерты.

**ВАЖНО:** визуал и визуальный стиль.

---

## Процесс работы

**Короткие итерации:**
- Начать с быстрых концептов (по 1–2 часа на каждый)
- Показать для feedback
- Доработать выбранное направление

Не тратить несколько дней на полировку всех вариантов. Лучше быстро показать черновые идеи и выбрать направление.

---

## Сценарий использования

Типичный пользователь открывает приложение и видит свой сохранённый контент:
- 5 рецептов от подруги (ссылки на food-блоги)
- 3 фото блюд из ресторанов
- 8 товаров (одежда, техника) с Amazon/магазинов
- 2 видео с YouTube
- 4 текстовые заметки

Пишет в чат: «Что приготовить на ужин для гостей?». Pinboard фильтруется, показывая только рецепты и фото еды. AI отвечает, используя эти сохранения.

Задача визуала — сделать так, чтобы этот экран хотелось открывать. Чтобы он выглядел как продукт, за который платишь $20–30/мес, а не как собранный на коленках проект.
